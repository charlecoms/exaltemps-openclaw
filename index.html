Mission reçue de Juli1 et professeur FX: Devenir un AI Architect

OPENCLAW

Keskécé

Openclaw n'est pas un simple bot. C'est un orchestrateur agentique LLM-native conçu pour interfacer vos prompts avec votre infrastructure réelle. Il peut coder, commit, monitorer et s'auto-réparer. En gros, c'est le stagiaire DevOps qui ne dort jamais, ne râle jamais, mais qui peut tout casser si vous ne le dressez pas correctement.

Pourquoi on fait ce TP (La réalité du terrain)

Incident #1: L'incident de la suppression massive d'emails (Mars 2026)

Un utilisateur a configuré OpenClaw pour gérer sa boîte de réception. Suite à une saturation de la mémoire contextuelle, l'outil a utilisé son mécanisme de "compaction" (résumé automatique pour libérer de la place). Durant ce processus, l'instruction de sécurité "Ne jamais supprimer d'emails non lus" a été effacée du contexte. La catastrophe: L'IA a commencé à supprimer de manière autonome des milliers d'emails, considérant la tâche comme autorisée. L'utilisateur a dû forcer l'arrêt physique de la machine pour stopper le massacre.
Voir l'article

Incident #2 : Le détournement par "Prompt Injection" via Email

En envoyant un email ou un message de chat contenant une instruction malveillante invisible, un attaquant a pu forcer l'assistant OpenClaw à extraire des fichiers internes, des identifiants de connexion et des historiques de conversations privées pour les envoyer sur un serveur externe. Le bot ne distinguait pas les ordres de son propriétaire de ceux contenus dans les messages qu'il lisait.

Votre mission: orchestration as prompt

01. Openclaw Gateway

L'accès via l'URL Azure est actuellement verrouillé par un token. Votre mission : le trouver dans la VM.

  • Prérequis : Git Bash (ou terminal SSH), compte GitHub, Application Telegram.
  • Cible : openclaw-exalt-team-[id]-90cd03b8.francecentral.cloudapp.azure.com
  • Accès VM : Connectez-vous avec votre clé SSH :
ssh -o StrictHostKeyChecking=no -i ~/.ssh/consultant azureuser@[IP]

Indice : Une fois dans la VM, cherchez où la Gateway cache son secret de connexion...

02. Dressage de l'Agent

Une fois sur l'interface, faites connaissance avec votre nouvel assistant. Ne soyez pas timide, mais soyez précis.

  • • Identité : Configurez son nom et imposez-lui un caractère d'expert DevOps (bonnes pratiques obligatoires).
  • • Mémoire : Configurez-le pour sauvegarder vos discussions quotidiennement dans des mémos horodatés.
  • • Méthode : Utilisez le prompt ou l'onglet "Agent". Plus le prompt est carré, moins vous galérerez.
  • • Bio : Dites-lui qui vous êtes, votre nom et vos habitudes de travail.

03. Interconnexions (Telegram & GitHub)

Sync Telegram

Configurez Telegram uniquement en prompt. Demandez-lui : "Envoie-moi 'coucou' sur Telegram". S'il ne le fait pas, recommencez.

GitHub Handshake

Connectez votre compte via le Device Login. Créez un token d'accès avec le Strict Minimum de droits nécessaires.

Automation, Cron & Vibecoding

"Look Mom, no hands!"

04a. Vibecoding & GitHub Pages

Faites coder un front Single Page (marrant ou pas) à Openclaw, demandez-lui de le push sur un nouveau repo public et d'activer les GitHub Pages. Vérifiez l'URL.

04b. La Sauvegarde Paranoïaque (Cron)

Demandez une synchronisation de toute son arborescence locale vers un repo PRIVÉ toutes les 5 min. Interdiction formelle de leaker une clé d'accès dans le repo !

04c. Healthcheck & Auto-Healing (Le Graal)

Créez un cronjob qui vérifie si votre site est online. Si ça tombe : notification Telegram + l'agent doit faire un commit automatique pour forcer le redeploy de GitHub Pages.

Security Audit & Feedback (Bonus)

Cet exercice est volontairement une "passoire" par endroits. S'il vous reste du temps, identifiez les failles (gestion des tokens, stockage des clés, droits SSH) et proposez des correctifs.

Rapports d'incidents (REX)

Incident #742 - The DB Wipe

Dans une scale-up, un agent a interprété une commande de "nettoyage de logs" comme une instruction de purge complète d'un cluster RDS. 4h de downtime.

Incident #109 - Git Bombing

Lors d'un conflit de merge, l'agent a généré 4 000 commits en 10 minutes, entraînant le bannissement temporaire du compte GitHub de l'entreprise.

Parcours Technique

01. Infiltration et Accès

L'accès via l'URL Azure est actuellement verrouillé. Vous devez extraire le token de la gateway manuellement.

  • Prérequis : Git Bash (SSH), compte GitHub et Telegram opérationnel.
  • Connectez-vous à la VM cible :
ssh -o StrictHostKeyChecking=no -i ~/.ssh/consultant azureuser@[IP]

02. Configuration Agentique

Définissez l'identité de votre assistant. L'agent doit apprendre qui vous êtes et comment travailler selon vos standards.

Configuration requise :

  • • Nom, caractère et bonnes pratiques DevOps de l'agent.
  • • Crucial : Activation de la sauvegarde des discussions par mémos quotidiens horodatés.
  • • Présentation de votre profil (nom, habitudes de travail).

03. Communications & Hub

Telegram Sync

Configurez l'intégration uniquement via prompt. L'agent doit confirmer par un "coucou" sur votre mobile.

GitHub Device Flow

Connectez votre compte. Restreignez les droits du token au strict nécessaire pour cette mission.

Automation & Cron

Phase finale : Autonomie complète

Vibecoding

Demandez à Openclaw de coder une Single Page, de créer un repo et d'activer GitHub Pages automatiquement.

Sync Cron

Synchronisation automatique de l'arborescence locale vers un repository privé toutes les 5 minutes.

Healthcheck & Auto-Healing

Monitoring de la page publique avec alerte Telegram. En cas de fail, l'agent doit forcer un commit pour déclencher un redeploy.

Security Audit & Feedback

L'exercice comporte volontairement des failles de sécurité. Identifiez les vulnérabilités (tokens, droits, stockage des clés) et proposez des solutions de durcissement.
